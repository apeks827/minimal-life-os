# Privacy Model

LifeInbox captures sensitive life data. The default posture is data minimization:

- AI memory storage must remain opt-in through settings.
- Raw inbox text should not be logged or sent to analytics.
- Provider prompts should include only the context needed for classification/planning.
- Memory reset/export/delete must cover raw inbox, derived entities, suggestions, memories/vectors, and settings.
- Retention windows and legal copy are not production-ready yet.

Production launch requires terms/privacy placeholders, retention policy, age/privacy assumptions, and explicit user copy for AI memory behavior.
